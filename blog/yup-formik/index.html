Don't overcomplicate forms (Reactjs).

While I was working on a project where I needed dynamic switches between forms and having them validated. I was thinking to myself, ok let us see what's a good way to validate the form and I stumbled upon yup. While it has a simple and cute name under the hood it's super fast and well built. It was perfect now I could have written the validation on my own but that would come at a cost of time witch I didn't have, and also more code. After that, I was searching for a good form helper and stumbled uponÂ Formik. What separates formik from the rest of the helpers is the performance( small library ) and how it handles the annoying parts of building a form component (Handling form submission,validation and error messages, getting values in and out of form state )

To install Formik and yup

NPM

npm install formik --save

npm install yup --save

Getting started

First, we need to import formik in our form component specifically we are gonna use a custom hook from formik called useFormik. It will return all states and helpers directly, If your trying to access the formik state via context , use useFormikContext , also don't use <Formik> or withFormik with useFormik() .

Now while we are at it, I will describe the form state in few words. The data we are receiving is for products that can be either book, furniture, or CD. The rest of the state goes as follows :

  • SKU (the stock-keeping unit)

  • Price(the price of a product)

  • Values (size or weight of the product )

  • Type (type of product e.g furniture, DVD or book)

Now here is where the stuff gets interesting , this is where we set the data types and requirements for the validations . Now, isFurniture is used as a boolean for triggering the type of value to be expected because we can't have furniture in MB. For this conditional validation, we used .when() and it accepts 2 parameters the name of the initial value(isFurniture) its checking, and object with specific rules.

  • is:true - is the condition in this example isFurniture===true

  • then and otherwise - is triggered when the condition is correct or false (you can think of it like try-catch or if-else )
